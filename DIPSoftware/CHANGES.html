ImprovementDIP-133DIP Browser JNLP must be available during TN-GPN cut<p>Current DIP Browser JNLP file (<a href="http://cern.ch/dipbrowser/launch.jnlp" class="external-link" rel="nofollow">http://cern.ch/dipbrowser/launch.jnlp</a>) is served by GPN web server. ImprovementDIP-131Ship Windows LIB binaries with the DIP API<p>The build does not include the Windows LIB file for the DIP library, but this is required for implicit linking (explicit linking is too much trouble for most users).</p>

ImprovementDIP-128Digitally sign the DIP JNLP packages with a CERN CA approved certificate<p>Due to new Java security restrictions, we need to sign the DIP JNLP packages with a CERN CA certificate, for it to be accepted by CERN computers that have the CERN CA available.</p>
ImprovementDIP-127Add Java system property to specify a default DIP NS to the DIP Browser<p>A Java system property should be available to pass a default DIP NS hostnames list to the DIP Browser.</p> <p>Another system property should force the DIP Browser to connect immediately (without waiting for the user to press the connection button).</p>

TaskDIP-125Upgrade to Java 1.7<p>The DIP Java API now requires Java 1.7 or later.</p>
New FeatureDIP-124DipData should expose name value pairs as java Map<p>DipData should expose its name value pairs as a standard java map.</p>
New FeatureDIP-123Add support for mock testing in the API<p>The Java API should expose basic mock objects (factory, publication, subscription, DipData) that would ease the unit testing of DIP related logic.</p> <p>In addition, a DipQualityEnum enumeration should be introduced to allow better programmatic handling of DIP quality.</p>
ImprovementDIP-122DIP Browser parses strings inefficiently and freezes<p>As identified in ENS-12060, the DIP Browser is very inefficient at turning large publication data arrays to string for display. Furthermore, the GUI thread is also the one in charge of parsing DIP data, which makes the whole application appear sluggish.</p> <p>We need to limit the parsing of arrays say to the first 100 entries, which is sufficient to indicate whether the publication contains valid data, but won't slow down the DIP browser.</p> <p>Initially reported in <a href="https://issues.cern.ch/browse/CMW-1352" class="external-link" rel="nofollow">https://issues.cern.ch/browse/CMW-1352</a></p>
